# Q30. I2C Protocol in IoT Sensor Communication

## What is I2C?

I2C (Inter-Integrated Circuit) is a communication protocol that allows multiple devices, such as sensors and microcontrollers, to communicate using only two wires. It is widely used in IoT systems because it reduces wiring complexity and supports multiple devices on the same communication bus.

## SDA and SCL Lines

I2C uses two communication lines:

- **SDA (Serial Data Line):** Transfers data between the microcontroller and connected devices.
- **SCL (Serial Clock Line):** Carries the clock signal generated by the master device to synchronize data transfer.

## How I2C Addressing Works

Each I2C device has a unique address (usually a 7-bit address). The master device (such as an Arduino) sends the address of the target device before transmitting or receiving data. Only the device with the matching address responds, while all other devices ignore the communication. This allows multiple sensors to share the same SDA and SCL lines without conflicts.

## Common IoT Sensors That Use I2C

1. **BMP280** – Temperature and barometric pressure sensor.
2. **MPU6050** – Accelerometer and gyroscope sensor.
3. **BH1750** – Digital light intensity sensor.

## Conclusion

The I2C protocol is a simple and efficient communication method for IoT devices. By using only the SDA and SCL lines and unique device addresses, multiple sensors can communicate with a single microcontroller using minimal wiring.
